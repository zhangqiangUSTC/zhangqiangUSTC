MY SERVICES

Key Value Store System

How to design a high-performance key-value store system on based on LevelDB and RocksDB. The goal is to reduce read and write amplification significantly and get efficient performance of insert and lookup

SSD Array System

How to design a data layout strategy on RAID Array System. The goal is to balance requests between SSD Array and improve the concurrent I/O on RAID Array System

LevelDB

LevelDB is a light-weight,single-purpose library for persistence with bindings to many platforms. It stores entries lexicographically sorted by keys which comes in very useful for querying

RocksDB

RocksDB is an embeddable persistent key-value store for fast storage.It uses a log structured database engine,keys and values are just arbitrarily-sized byte streams

LevelDB is a relatively new entrant into the growing list of key/value database libraries, but it has some very interesting qualities that we believe make it an ideal candidate for use in Riak. LevelDB’s storage architecture is more like BigTable’s memtable/sstable model than it is like Bitcask. This design and implementation provide the possibility of a storage engine without Bitcask’s RAM limitation.

The experiment about RocksDB

  • Admin
  • 2 Comments
  • 50 Likes
  • 3 Tags

The rocksdb project started at Facebook as an experiment to develop an efficient database software that can realize the full potential of storing data on fast storage ( especially flash storage) for server workloads. It has support for atomic reads and atomic writes. It has highly flexible configurable settings that can be tuned to run on a variety of production environments: it can be configured to run on data on pure memory, flash, hard disks or on HDFS.

The experiment about RAID

  • Admin
  • 2 Comments
  • 50 Likes
  • 3 Tags

Redundant Array of Independent Disks is a data storage virtualization technology that combines multiple physical disk drive components into one or more logical units for the purposes of data redundancy and performance improvement. Data is distributed across the drives in one of several ways, referred to as RAID levels, depending on the required level of redundancy and performance. The different schemes, or data distribution layouts, are named by the word "RAID" followed by a number, for example RAID-0 or RAID-1.
